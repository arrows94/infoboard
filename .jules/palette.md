
## 2024-03-04 - Accessible Custom Toggles and Label Targets
**Learning:** In the frontend admin UI, custom UI toggles visually replacing checkboxes used `display: none` on the original `input[type="checkbox"]`. This completely removes the element from the accessibility tree and prevents keyboard navigation (`Tab` key focus) entirely. Furthermore, using `<label>` elements without explicitly mapping them to their corresponding inputs with the `for` attribute misses a huge opportunity to provide a larger clickable area for form elements, making toggling and selecting difficult for users with motor impairments.
**Action:** When implementing custom inputs (like toggle switches), never use `display: none` on the native input. Instead, use `opacity: 0; position: absolute; width: 0; height: 0;` to keep it focusable by keyboard. Always add explicit `:focus-visible` styles on the custom element's sibling when the hidden input receives focus. Always ensure every `<label>` has a `for="[id]"` attribute that matches its input.
